# Analyst Assessment Framework

## Core Question
Does the activity reflect normal user behavior, or does it indicate potential unauthorized access?

## Factors to Evaluate
- Is the login device known or unknown?
- Is the login location expected?
- Did the user recently reset their password?
- Were MFA settings changed?
- Were account details changed after login?
- Did financial or sensitive actions occur soon after access?
- Does the timing align with the user's normal behavior?

## Indicators That Increase Suspicion
- Multiple authentication anomalies
- Account recovery changes
- Contact detail updates
- Immediate transaction activity
- New device plus unusual location
- Deviation from historical user behavior

## Indicators That Reduce Suspicion
- Known device and location
- User-confirmed password reset
- No sensitive changes after login
- Normal session pattern
- Expected login timing

## Analyst Assessment
The strongest account takeover cases usually involve a sequence of related anomalies rather than a single isolated alert.
